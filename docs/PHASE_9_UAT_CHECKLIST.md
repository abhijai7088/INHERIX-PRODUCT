# Phase 9 UAT Checklist

## UAT Goal
Confirm that the product matches the approved INHERIX workflow and is understandable to business users, admins, and trusted nominees.

## UAT Participants
- Customer / owner representative
- Nominee / beneficiary representative
- Admin operator
- Super admin reviewer

## Checklist
- [ ] Customer can register and authenticate
- [ ] Customer can create a vault
- [ ] Customer can upload a record and store metadata
- [ ] Customer can invite nominees
- [ ] Customer can define access rules
- [ ] Nominee can submit a trigger request
- [ ] Nominee can upload proof
- [ ] Admin can review proof and make a decision
- [ ] Admin can selectively release only eligible documents
- [ ] Nominee can only see released documents
- [ ] Audit logs appear for sensitive actions
- [ ] Notifications appear for workflow and security events
- [ ] Security wording and disclaimers are clear
- [ ] Screens match the approved wireframe language and style
- [ ] Error messages are understandable and non-technical
- [ ] Mobile behavior is acceptable
- [ ] Admin / governance / backup / report screens are understandable

## UAT Acceptance Criteria
- The core workflow feels trustworthy
- Terminology matches the product definition
- No unrestricted vault browsing is exposed
- Release only occurs after proper approval
- Security and audit visibility are clear
- No critical user-blocking defect remains open

## Signoff Notes
- UAT signoff should be recorded with date, participant, scope, and any accepted exceptions.
